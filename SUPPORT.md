# Support

OpenProof is proof-of-existence infrastructure in active prototype form. Support is best-effort and focused on reproducible proof, receipt, wallet, documentation, and local development issues.

## Before Opening An Issue

- Read the [README](README.md), [SECURITY.md](SECURITY.md), and [CONTRIBUTING.md](CONTRIBUTING.md).
- Search existing issues and pull requests.
- Do not upload private files, sensitive receipts, seed phrases, private keys, or wallet secrets.

## Good Issue Reports

- Describe the browser, wallet, network, and transaction state.
- Include receipt JSON only if it is safe to make public.
- Include transaction hashes for public testnet behavior when relevant.

For vulnerabilities or sensitive reports, follow [SECURITY.md](SECURITY.md).
